# lsearch TUI Checkbox Fix

## Problem
questionary.checkbox was throwing ValueError when using default value with Choice objects:
```
ValueError: Invalid \`default\` value passed. The value (\`['./docs']\`) does not exist in the set of choices.
```

## Root Cause
questionary.checkbox cannot match default values against Choice objects properly. The default value must be a string that exists in the choices list.

## Solution
Changed from using Choice objects to plain strings for checkbox choices.

## Changes Made
- src/lsearch/cli.py: Simplified checkbox to use plain strings
- All 36 tests passing
